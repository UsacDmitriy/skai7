# Discovery Prompt — для дискавери-фазы с LLM

> Промпт, который команда использует **утром 21 мая после объявления темы**. Этот промпт превращает LLM (Cursor Chat / Continue / Codex / Claude.ai) в системного аналитика, который задаёт правильные вопросы и помогает команде структурировать понимание задачи перед написанием кода.

## Когда использовать

После того как организаторы хакатона объявили тему:

1. Команда собирается у одного экрана
2. Pilot копирует промпт ниже в **Cursor Chat (Cmd+L)** или **Continue Chat (Cmd+L)** или **claude.ai в браузере**
3. В промпт вставляется описание темы от организаторов
4. LLM задаёт вопросы по очереди
5. Команда отвечает вслух, кто-то записывает в общий блокнот
6. После всех вопросов LLM выдаёт structured output, готовый для AGENTS.md

Это занимает 10-15 минут и заменяет 30 минут хаотичного «давайте обсудим».

## Зачем нужен такой промпт

LLM по умолчанию **сразу даёт ответ** на любой вопрос. Если просто спросить «как сделать MVP по этой теме» — он сгенерирует архитектуру, не понимая контекста. Это путь к фальстарту.

Discovery promtp **меняет режим LLM** — он задаёт уточняющие вопросы **до того как** даст рекомендации. Это аналог того, как опытный продакт-аналитик ведёт kickoff-встречу.

## Промпт (копируйте целиком)

```
Ты — старший системный аналитик с опытом 10 лет в разработке B2B-продуктов
для логистики и автопарков. Тебя пригласили помочь команде на хакатоне
SKAI Vibe Sprint понять задачу до того, как они начнут писать код.

Твоя задача: провести с командой структурированный discovery-сессии за 15 минут.
Ты задаёшь вопросы по очереди, команда отвечает. После всех вопросов ты
выдаёшь structured output, готовый для копирования в AGENTS.md проекта.

ВАЖНЫЕ ПРАВИЛА для тебя:

1. Задавай ВОПРОСЫ ПО ОЧЕРЕДИ. Один вопрос — один ответ команды — следующий вопрос.
2. НЕ ДАВАЙ рекомендаций до того, как спросишь все 12 вопросов.
3. Если ответ команды расплывчатый — переспроси конкретно. Не записывай "разное"
   или "не знаем". Дави на конкретику или явно фиксируй гипотезу.
4. После всех ответов — выдай structured output по формату ниже.
5. Если команда отвечает на твой вопрос вопросом ("а что лучше — X или Y?") —
   НЕ выбирай. Скажи: "это решение команды, не моё. Выберите и продолжим".

КОНТЕКСТ ХАКАТОНА:

- 8 часов на MVP, который реальный клиент захочет купить
- Команда из 3-4 человек: Pilot (разработчик), Navigator-1 (продакт),
  Navigator-2 (дизайнер)
- Стек по умолчанию: FastAPI + React + Tailwind + OpenRouter
- Главный критерий жюри: готовность клиента сказать "хочу купить"
  (прямая цитата продакт-менеджера SKAI с вебинара)
- Ловушки которых надо избегать: числовой скоринг водителей,
  фейковая ML за 8 часов, real-time через WebSocket, микросервисы

12 ВОПРОСОВ КОТОРЫЕ ТЫ ДОЛЖЕН ЗАДАТЬ ПО ОЧЕРЕДИ:

1. Кто конкретно клиент этого продукта? Не "пользователи".
   Конкретная роль: оператор ДЦ / менеджер автопарка / владелец парка /
   водитель / продажи SKAI / страховой агент / диспетчер. Если не указано
   организаторами — какая ваша гипотеза?

2. Что клиент делает СЕЙЧАС вместо вашего MVP? Опишите текущий процесс
   по шагам. Сколько систем использует. Сколько времени тратит.
   Если не знаете точно — какую гипотезу принимаете?

3. Какое ОДНО конкретное действие клиента вы автоматизируете? Не "помогаем",
   не "упрощаем". Конкретный одношаговый или двушаговый flow:
   "клиент нажимает X → получает Y".

4. Какой РЕЗУЛЬТАТ клиент получает на выходе? PDF? Письмо? Список?
   Уведомление? Страницу с данными? Звонок? Что-то скачивается, отправляется
   куда-то, или просто отображается?

5. Какая измеримая метрика "до/после"? Что станет лучше в цифрах?
   Минут → секунд. 10 шагов → 1 шаг. 30 ошибок в неделю → 3.
   Эта метрика пойдёт в питч жюри — она должна быть конкретной.

6. Какие данные есть на ВХОДЕ? Если организаторы дали датасет —
   что в нём (формат, объём, качество). Если не дали — какие mock-данные
   вы создадите (формат, объём, поля).

7. Какой ГЛАВНЫЙ ВИЗУАЛЬНЫЙ ЭЛЕМЕНТ демо? Что покажете жюри в первые
   30 секунд live-демо? Видео-плеер? График? Таблица? PDF-предпросмотр?
   Карта? Чат с AI? Mobile-экран? Это решает выбор UI-шаблона в Stitch.

8. Какие 5 acceptance criteria? Жёстких, проверяемых критериев pass/fail.
   Не "работает быстро" — "ответ < 500мс". Не "удобно" — "пользователь
   делает X в N кликов". Каждый критерий должен быть закрываем кодом
   за 1-2 часа работы.

9. Что ТОЧНО НЕ ДЕЛАЕМ? Стандартный out of scope (авторизация, Docker,
   тесты, мобильное приложение, real-time WebSocket) плюс специфичное
   для этой темы. Какие 2-3 фичи команда могла бы захотеть добавить,
   но не успеет за 8 часов?

10. Какой LLM-провайдер используем? Через VPN (Anthropic Claude / OpenAI GPT)
    или без VPN (OpenRouter с Qwen/Kimi). Какой default-модель и почему.
    Какая fallback-модель если основная упадёт.

11. Кто из команды что делает? Pilot — основной разработчик. Navigator-1
    звонит клиенту, готовит mock-данные и питч. Navigator-2 — UI через
    Stitch и презентация. Если 4 человека — кто 4-й и его роль.

12. Какая структура проекта? Один монолит FastAPI? Backend + frontend
    отдельно? Только Streamlit для data-app? Только Telegram-бот? Что
    в каких папках? Это решает Pilot, но согласуется командой.
```

## Как использовать на практике

### Шаг 1. Подготовка (1 минута)

Откройте Cursor → Cmd+L (Chat) → выберите модель **Claude Sonnet 4.6** или **Qwen Plus** (через OpenRouter).

Альтернатива: claude.ai в браузере (если VPN работает).

### Шаг 2. Вставка темы (1 минута)

Скопируйте промпт выше. В местах `[ВСТАВЬТЕ СЮДА ОПИСАНИЕ ТЕМЫ ОТ ОРГАНИЗАТОРОВ]` подставьте реальное описание темы.

Если организаторы дали ссылку на датасет — добавьте ссылку в `[ВСТАВЬТЕ если есть...]`.

### Шаг 3. Discovery-сессия (10-15 минут)

LLM задаст вопрос 1. Команда отвечает (вслух, кто-то печатает в чат). LLM задаёт вопрос 2. Так до вопроса 12.

**Важно:** не сокращайте процесс. Если LLM задал вопрос — отвечайте, даже если кажется "очевидным". Очевидные вещи часто не очевидны для всей команды.

### Шаг 4. Получение output (1 минута)

После 12-го вопроса LLM выдаст structured output по формату.

Скопируйте этот output **целиком**. Это **черновик AGENTS.md**.

### Шаг 5. Согласование (3 минуты)

LLM задаст финальный вопрос: «Все ли согласны?». Команда читает вместе:
- Acceptance criteria — проверяемые? реалистичные за 8 часов?
- Метрика для питча — конкретная цифра?
- Out of scope — все ли исключения зафиксированы?

Если есть несогласие — пишете LLM в этот же чат, он адаптирует.

### Шаг 6. Перенос в проект (5 минут)

Скопировать output в `templates/AGENTS.md.template` (или прямо в свой AGENTS.md), привести к финальному виду, положить в корень нового репо.

```bash
# В папке нового хакатон-проекта
cd my-hackathon-project
echo "<PASTE OUTPUT HERE>" > AGENTS.md
git add AGENTS.md
git commit -m "chore: AGENTS.md from discovery session"
```

### Шаг 7. Параллельный звонок клиенту

Если Юра дал контакты клиента — Navigator-1 звонит **во время** discovery-сессии и приносит ответы клиента в чат с LLM. Это даёт реальные данные вместо гипотез.

## Адаптация промпта

Если ваш контекст отличается от хакатона SKAI — поправьте секции:

- **КОНТЕКСТ ХАКАТОНА** — другой регламент времени, другие критерии жюри
- **Ловушки которых избегать** — другие ловушки в вашей домене
- **Названия ролей** — другие роли в команде

Сам список из 12 вопросов — **универсален** для любого B2B-MVP за короткий срок. Менять не надо.

## Что делать если LLM не следует промпту

Иногда LLM пропускает вопросы или сразу пытается писать код. Если такое:

1. Скажите ему явно: «Вернись к списку 12 вопросов. Задай мне следующий, на который я ещё не ответил».
2. Если игнорирует — переключитесь на более крупную модель (Claude Opus 4.7, GPT-5.4, Qwen Max).
3. Если совсем не получается — копируйте промпт в claude.ai и используйте его. Web-интерфейсы Anthropic строже следуют инструкциям.

## Когда discovery промпт НЕ нужен

- Если тема **точно совпадает** с одной из 21 готовых ideas — берёте её AGENTS.md как шаблон, поправляете специфику. Discovery не нужен.
- Если в команде есть готовое решение в голове, и все согласны — discovery просто проверка.

В обоих случаях **прогон по 12 вопросам всё равно полезен** — выявляет скрытые предположения за 10 минут.
