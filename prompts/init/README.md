# Prompts/

> Готовые промпты для хакатона SKAI: «Единое окно для видео и телематики». Целевой стек — Streamlit (Python).

## Когда какой промпт

| Когда | Промпт | Кто использует |
|---|---|---|
| Утром, один раз за день хакатона (10–15 мин) | [`discovery-prompt.md`](./discovery-prompt.md) | Вся команда вместе |
| Каждые 30–60 мин во время работы (5–10 мин на запуск) | [`orchestration-prompt.md`](./orchestration-prompt.md) | Pilot / Team Lead |
| Первый запуск репозитория (параллельно) | [`hackathon-init/`](./hackathon-init/) | Любой из команды |

---

## Discovery Prompt

**Назначение:** превратить тему хакатона в структурированное боевое задание за одно утреннее интервью.

LLM выступает в роли **системного аналитика**. Задаёт **12 вопросов** по порядку, дожидается ответа команды на каждый, затем сводит всё в готовый `AGENTS.md`.

### Порядок вопросов

| № | Тема вопроса | Пример ожидаемого ответа |
|---|---|---|
| 1 | **Заказчик** — кто потребитель результата, его роль и контекст | «Диспетчер транспортного парка, 200+ машин» |
| 2 | **Текущий процесс** — как задача решается сейчас, что не устраивает | «Диспетчер переключается между 3 окнами: система телематики, видеорегистратор, Excel» |
| 3 | **Одно ключевое действие** — главное, что должно происходить в системе | «Диспетчер видит инцидент с машиной в одном окне и принимает решение за 5 секунд» |
| 4 | **Результат** — измеримый outcome после внедрения | «Время реакции на инцидент сокращено с 2 минут до 30 секунд» |
| 5 | **Метрика** — как измерить успех, конкретный KPI | «Среднее время от инцидента до реакции диспетчера» |
| 6 | **Источники данных** — что за данные, откуда, в каком формате | «Телематика: JSON API (Wialon/Omnicomm). Видео: RTSP-потоки с бортовых камер. Архив: MP4-файлы» |
| 7 | **Ключевой визуальный элемент** — что видит пользователь в первую очередь | «Дашборд: карта с машинами, панель инцидентов, окно видеоплеера — всё на одном экране» |
| 8 | **Критерии приёмки** — 3-5 жёстких условий сдачи демо | «1. Карта с живыми точками машин. 2. При клике на инцидент открывается видео за 3 сек. 3. Телематические графики рядом с видео.» |
| 9 | **Вне скоупа** — что явно НЕ делаем на хакатоне | «Мобильное приложение, авторизация, real-time видеоаналитика, свой видеоплеер с нуля» |
| 10 | **LLM-провайдер и модель** — какой AI используется в системе | «OpenAI GPT-4o через proxy для суммаризации инцидентов» |
| 11 | **Роли в команде** — кто за что отвечает | «Pilot: архитектура + оркестрация. Backend: FastAPI + телематика. Frontend: Streamlit. QA: тесты + данные. AI: интеграция LLM» |
| 12 | **Структура проекта** — соглашение о папках и именовании | «`backend/`, `frontend/`, `shared/`, `tests/`, `data/`, `docs/`» |

### Output

После 12 вопросов LLM формирует файл `AGENTS.md` со следующими секциями:
- **Резюме проекта** (2-3 абзаца)
- **Текущий процесс и боль**
- **Одно ключевое действие**
- **Результат и метрика**
- **Источники данных**
- **Ключевой визуальный элемент**
- **Критерии приёмки демо**
- **Вне скоупа**
- **LLM-провайдер**
- **Роли команды**
- **Структура репозитория**
- **Определение Done для демо**

## Orchestration Prompt

**Назначение:** держать архитектурную целостность проекта в условиях параллельной работы нескольких агентов (или разработчиков).

LLM выступает в роли **главного архитектора**. Каждые 30–60 минут получает на вход:
- Текущий `AGENTS.md`
- Кусок кода/логов/диффов от каждого члена команды (или агента)
- Статус критериев приёмки с прошлой итерации

### Расписание запусков

| Время | Назначение |
|---|---|
| **11:30** | Утренняя сверка после discovery, первые стримлит-заглушки |
| **13:00** | Первая интеграционная проверка: backend ↔ frontend |
| **14:30** | Середина дня: API + видео + телематика должны стыковаться |
| **16:00** | Предфинальная проверка: все acceptance criteria |
| **17:00** | Финальная полировка: демо-скрипт, стабильность, документирование |

### Формат отчёта

```
## Состояние проекта
<текущий статус: что работает, что в процессе, что блокировано>

## Несоответствия
<конкретные интерфейсные/контрактные расхождения между модулями>
Пример: «Backend отдаёт поле 'timestamp' как Unix-ms, фронтенд ожидает ISO-8601»

## Статус критериев приёмки
| Критерий | Статус | Комментарий |
|---|---|---|
| Карта с живыми точками | ✅ Выполнено | |
| Инцидент → видео за 3 сек | ⚠️ В процессе | Скорость загрузки 7 сек, нужен кеш |
| ... | | |

## Оценка рисков
- **Красный** — блокирует демо
- **Жёлтый** — замедляет, но обходимо
- **Зелёный** — под контролем

## Action Items по ролям
| Приоритет | Роль | Задача | Дедлайн |
|---|---|---|---|
| P0 — блокирует демо | Pilot | ... | ... |
| P1 — обязательно к демо | Backend | ... | ... |
| P2 — желательно | Frontend | ... | ... |
| P3 — если время останется | QA | ... | ... |
```

### Правила оркестратора

- Основывается только на реальных диффах и логах — не фантазирует
- Если два модуля расходятся в интерфейсе — указывает конкретную строку/поле и предлагает единый контракт
- Не пишет код — только direction и action items
- Эскалирует P0-блокеры немедленно (а не ждёт следующего раунда)

## Hackathon-init

**Назначение:** за 1 заход развернуть полную инфраструктуру репозитория для Streamlit-приложения хакатона SKAI.

Четыре параллельных промпта в папке `hackathon-init/`:

### 1. Git + Codex / MCP Config

Инициализирует:
- `git init` + стандартный `.gitignore` (Python, venv, IDE, env-файлы)
- Файл `kilo.json` / `mcp.json` с MCP-серверами (filesystem, postgres если нужно, context7)
- Правила ветвления: `main` защищён, фичи через feature-ветки или worktree

### 2. Backend (FastAPI stubs)

Инициализирует:
- Папку `backend/` с FastAPI-приложением
- Роуты-заглушки: `/api/telematics`, `/api/incidents`, `/api/video/stream`
- Модели Pydantic (TelematicsEvent, Incident, VideoSource и т.д.)
- `requirements.txt` backend-зависимостей (fastapi, uvicorn, httpx, openai, pydantic)

### 3. Frontend → Streamlit Setup

Инициализирует:
- Python venv в корне проекта: `python3 -m venv venv`
- `requirements.txt` с зависимостями: `streamlit`, `folium` (карты), `plotly` (графики), `opencv-python-headless` (видео), `httpx`, `pandas`, `python-dotenv`
- Папку `frontend/` со скелетом Streamlit-приложения:
  - `app.py` — точка входа с layout'ом страницы, сайдбаром, местом под карту, видео и инциденты
  - `components/map.py` — заглушка карты (folium + streamlit-folium)
  - `components/video.py` — заглушка видеоплеера
  - `components/incidents.py` — заглушка панели инцидентов
  - `components/telemetry.py` — заглушка графиков телематики (plotly)
- `run.sh` — скрипт запуска: активация venv, `streamlit run frontend/app.py`

### 4. QA Integration

Инициализирует:
- Папку `tests/` с `conftest.py`, `pytest.ini`, `pyproject.toml`
- Smoke-тесты: проверка что `app.py` импортируется, что API-ручки отвечают 200 на `GET /health`, что компоненты рендерятся без ошибок
- Фикстуры: тестовые данные телематики, инцидентов, конфигурации
- `scripts/generate_test_data.py` — генератор синтетических телематических данных и инцидентов для разработки и тестов

## Принципы написания промптов

Все промпты в этой папке построены по единой модели:

```
Роль → Правила → Контекст → Слоты → Структурированный Output → Пост-поведение
```

| Элемент | Описание |
|---|---|
| **Роль** | Чёткая персона LLM с экспертными границами (аналитик, архитектор, fullstack-разработчик) |
| **Правила** | Что LLM делает и что явно не делает, формат ответа, ограничения |
| **Контекст** | Информация о хакатоне, теме («Единое окно для видео и телематики»), стеке (Streamlit, FastAPI, Python) |
| **Слоты** | Места `{{}}` для вставки специфики командой перед запуском (тема дня, путь к AGENTS.md, диффы агентов) |
| **Структурированный Output** | Жёсткий формат ответа — LLM не может отклониться от структуры |
| **Пост-поведение** | Что происходит после выдачи ответа: запись в файл, следующий раунд, эскалация |

## Что ещё

- В `templates/AGENTS.md.template` — шаблон, который заполняет Discovery Prompt
- В `templates/prompt-codex.md.template` — стартовый промпт для запуска агента на конкретной задаче
- В `playbook/` — техники стыковки модулей, примеры промптов для видеоинтеграции и работы с телематическими API

### Жизненный цикл промптов на хакатоне

```
Discovery (prompts/init/discovery-prompt.md)        ← 1 раз утром
  ↓
AGENTS.md заполнен                                   ← результат discovery
  ↓
hackathon-init/ (4 параллельных промпта)             ← репо готово за 10 мин
  ↓
Работа по ролям: backend, frontend, QA, AI          ← параллельная разработка
  ↓
Orchestration (prompts/init/orchestration-prompt.md)  ← каждые 30–60 мин
  ↓
Финальная полировка + демо-скрипт                    ← ~17:00
```
