# Промпт 3 — Streamlit UI

Задача: настроить и проверить Streamlit UI-скелет приложения «Единое окно видео и телематики».

Рабочая директория: `05_video_telematics_single_window/`

## 1. Проверка `app/app.py`

Файл `app/app.py` должен содержать:

```python
st.set_page_config(page_title="SKAI Hackathon MVP", layout="wide")
```

Три вкладки в `st.tabs`:

- **«Данные»** (бывшая data_tab) — загрузка и отображение CSV-файлов из автоопределяемого источника.
- **«Дашборд»** (бывшая dashboard_tab) — KPI-метрики: количество CSV-файлов, строк, событий-кандидатов, статус MVP.
- **«Детали»** (бывшая details_tab) — расширенная таблица рисков (top-20) с русскими метками и цветовой индикацией.

Автоопределение источника данных:

- Если в `data/` есть `*.csv` — используется `data/`.
- Иначе — `sample_data/`.

Сайдбар:

- Показывает путь к источнику данных.
- Содержит форму «Действие диспетчера» (ID записи, тип действия, комментарий, кнопка сохранения).

## 2. Проверка `app/constants.py`

Файл должен содержать:

- `COLORS` — словарь цветов (primary, critical, high, warning, medium, ok, low, info, chart_colors).
- `ALARM_TYPE_LABELS` — русские метки типов алармов (Drowsiness → Засыпание, и т.д.).
- `ACTION_TYPES` — список: `["mark_reviewed", "create_task", "export_report"]`.
- `ACTION_LABELS` — русские подписи к типам действий.
- `PAGE_TITLE = "SKAI Hackathon MVP"`.
- `PAGE_ICON = "SKAI"`.
- `LAYOUT = "wide"`.
- `RISK_THRESHOLD = 70`.
- `DATA_DIR_NAME`, `SAMPLE_DATA_DIR_NAME`, `OUTPUT_DIR_NAME`.
- `KPI_LABELS` — русские подписи метрик дашборда.

## 3. Проверка вспомогательных модулей `app/`

Убедись, что существуют и импортируются без ошибок:

| Модуль | Назначение |
|--------|------------|
| `data_loader.py` | `load_csv_files(data_dir)` — читает все CSV рекурсивно; `save_action(...)` — пишет в `output/actions.csv` |
| `metrics.py` | `build_dashboard_metrics(datasets)` — 4 KPI; `build_risk_table(datasets)` — top-20 рисков |
| `risk_table.py` | `render_risk_table(datasets)` — таблица с русскими метками и цветами |
| `charts.py` | `build_speed_chart(df)`, `build_track_map(df)`, `build_scatter_chart(df)` — Altair-графики |
| `actions.py` | `render_action_form(output_dir)` — форма действий в сайдбаре |

## 4. Запуск и проверка

```bash
source .venv/bin/activate
streamlit run app/app.py
```

Ожидаемое поведение:

- Приложение стартует на `http://localhost:8501`.
- Три вкладки видны: «Данные», «Дашборд», «Детали».
- Вкладка «Данные» отображает список CSV-файлов и их содержимое.
- Вкладка «Дашборд» показывает 4 KPI-метрики (CSV файлов, Строк загружено, Событий-кандидатов, Статус MVP).
- Вкладка «Детали» показывает таблицу рисков top-20 с колонками: id, vehicle_id, Тип события, risk_score, Уровень риска, reason.
- Сайдбар показывает источник данных и форму действия диспетчера.
- Консоль браузера не содержит ошибок JavaScript.

## 5. Устранение типовых проблем

Если вкладки пустые — проверь, что в `data/` действительно есть CSV-файлы с ожидаемыми колонками (`risk_score` или `severity`).

Если KPI показывает нули — проверь имена колонок в датасетах, они должны совпадать с теми, что ищет `metrics.py`.

Если импорт модулей падает — убедись, что запуск идёт из корня проекта (где лежит `app/`), а не изнутри `app/`.

## Запрещено

- Удалять существующие модули `app/`.

Streamlit-приложение запускается на `:8501`, три вкладки наполнены данными, KPI и таблица рисков отображаются корректно. Форма действия диспетчера сохраняет записи в `output/actions.csv`.
