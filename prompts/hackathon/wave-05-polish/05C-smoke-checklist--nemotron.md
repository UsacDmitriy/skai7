# Волна 5-C — Ручной smoke-test чеклист (Streamlit)

> 🤖 **Модель: `nvidia/nemotron-3-super-120b-a12b:free`** | проверка логики
> 💰 **Бюджет:** читать max 1–3 файла · не читать `./data`, `.venv`, `node_modules` · логи: 20–50 строк
> ⚠️ **Один промпт = одна сессия Cherry Studio.** Не запускать параллельно.

## Задача

Создать документ с чеклистом ручного smoke-тестирования Streamlit-приложения «Единое окно видео и телематики».

**Выходной файл:** `docs/SMOKE_CHECKLIST.md`

**Никакого кода. Только чеклист-документ.** Агент должен создать готовый Markdown-файл с полным списком проверок для ручного тестирования.

## Содержание чеклиста

Документ должен содержать следующие разделы и пункты проверки:

### 1. Вкладка «Данные» (Data tab)

- [ ] Приложение запускается: `streamlit run app/app.py`
- [ ] Отображается таблица «Загруженные CSV файлы» с колонками: file, rows, columns
- [ ] В боковой панели есть селектор «Датасет»
- [ ] При выборе датасета отображается превью (первые 100 строк)
- [ ] Если CSV-файлы отсутствуют — показывается предупреждение
- [ ] Данные из вложенной папки `work_rest_single_vehicle/` загружаются

### 2. Вкладка «Дашборд» (Dashboard tab)

- [ ] Отображаются 4 KPI-метрики: CSV файлов, Строк загружено, Событий-кандидатов, Статус MVP
- [ ] Значения метрик корректны (не нулевые при наличии данных)
- [ ] Селекторы осей X и Y работают
- [ ] Altair-график отображается при выборе осей
- [ ] При наличии колонок `timestamp_utc` + `speed_kmh` показывается график скорости
- [ ] При наличии колонок `latitude` + `longitude` показывается трек маршрута

### 3. Вкладка «Детали» (Details tab)

- [ ] Отображается таблица рисков (топ-20)
- [ ] Типы событий отображаются на русском языке
- [ ] Уровни риска раскрашены (🔴 Критический, 🟠 Высокий, 🟡 Средний, 🔵 Низкий)
- [ ] Форма «Действие диспетчера»: поле ID, селект действия (Проверено / Запросить видео / Создать отчёт), комментарий
- [ ] При отправке формы: данные сохраняются в `output/actions.csv`
- [ ] Показывается сообщение об успехе

### 4. Общие проверки

- [ ] Все три вкладки переключаются без ошибок
- [ ] В консоли нет Python traceback
- [ ] Все надписи на русском языке
- [ ] Приложение работает оффлайн (только локальные CSV и media)
- [ ] Нет импортов production-сервисов SKAI

### Итог

- [ ] 0 ошибок в консоли
- [ ] Все импорты разрешаются
- [ ] Данные отображаются корректно
- [ ] Если есть ❌ — записать что именно не работает и в каком файле искать

## Требования к документу

1. Документ на русском языке.
2. Каждый пункт — это checkbox Markdown (`- [ ]`).
3. В начале документа — инструкция для тестировщика: как запустить приложение и с чего начать.
4. В конце — секция «Результаты» с полями: Дата, Тестировщик, Общий результат (пройден / не пройден), Замечания.
5. Без кода, без промптов для других моделей — только готовый чеклист для человека.
