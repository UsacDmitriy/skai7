## Сценарий демо (4 минуты)

### 00:00–00:30 — Проблема

Опиши проблему:
- Диспетчер сегодня работает в 3 системах: телематика, видеоархив, заявки
- 5–7 минут на один инцидент
- Нет единого окна, где событие, трек, видео и действие в одном месте

### 00:30–01:00 — Наш ответ

Опиши решение:
- SKAI Unified Window: одно окно = полный контекст
- Стек: Python + Streamlit, оффлайн, 3 вкладки
- Показываем запуск: `streamlit run app/app.py`

### 01:00–02:00 — Вкладка «Данные» + «Дашборд»

Показываем:
- Data tab: 7+ CSV файлов, обзор, превью любого датасета

- График скорости: превышение на треке
- Карта маршрута: видно, где произошло событие

### 02:00–03:00 — Вкладка «Детали»

- Открываем `output/actions.csv` — демонстрируем, что заявка записана

### 03:00–03:40 — Ценность и метрики

Проговариваем цифры:
- Время на инцидент сокращается с 5–7 до 1–2 минут
- Одно окно вместо трёх систем
- Локальный оффлайн-прототип — работает без интернета
- Данные реальные: 55 алармов, 22 ТС, 6636 точек трека

### 03:40–04:00 — Закрытие

Завершаем:
- Готово к расширению: видео-превью, real-time стриминг

## Резервные ответы на вопросы жюри

Добавь в PITCH.md блок с ответами:

**Q: Почему Streamlit?**
> Быстрый прототип, 8 часов, фокус на ценности, а не на интерфейсе.

**Q: Где видео?**
> `media_relative_path` в данных, превью добавляется через `st.video()`.

**Q: Как матчинг телематики и видео?**
> По `alarm_id` + временное окно ±30 секунд.

**Q: Планы?**
> Карта Leaflet, видео-плеер, real-time WebSocket.
