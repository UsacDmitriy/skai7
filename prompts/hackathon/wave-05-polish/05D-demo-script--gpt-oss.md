## Структура питча (4 мин + 2 мин вопросы)

### 00:00–00:30 — Проблема (1 слайд)
> «39 ДТП — видеоаналитика помогла расследовать только 5.
>  Потому что видео и телематика в разных системах.»

### 00:30–01:00 — Наш ответ
Открыть :5173.
> «Вот что мы сделали — единое окно, где событие → доказательство → действие.»

### 01:00–02:00 — Flow 1: ДТП без DMS (KILLER DEMO)
1. В ленте нажать на **inc-002** (В345КМ 97, Петров, красная граница).
2. Показать TopBar: «Датчик удара сработал, DMS не сработал — боковое столкновение.»
3. Показать TelemetryChart: скорость 54 → 0, акселерометр пик 4.2 м/с².
4. Сказать: «Без единого окна — только телематика. Теперь → видео + данные + контекст.»
5. Нажать **"🔧 Создать заявку"** → кнопка disabled "✓ Заявка создана".
6. Сказать: «Действие зафиксировано. Никакой ручной работы.»

### 02:00–02:40 — Flow 2: Кейс B — нет видео
1. В ленте нажать на **inc-003** (Сидоров, чип "Нет видео").
2. Показать VideoEvidencePanel: плейсхолдер + кнопка "Запросить архив".
3. Показать ActionPanel: CAM-03 красный "Нет сигнала".
4. Нажать "📋 Запросить архивное видео" → "✓ Запрос создан".
5. Сказать: «Система объясняет почему нет видео и запускает процесс.»

### 02:40–03:10 — Flow 3: NL-запрос по водителю (КЛЮЧЕВОЙ СЦЕНАРИЙ — Оздоев / ГПН)
1. Перейти в «Диалог аналитики» в сайдбаре.
2. Нажать 🎤 и сказать вслух: «Покажи нарушения по ВА и телематике у Иванова за последние три дня»
   — Whisper AI расшифровывает, текст появляется в поле. (Альтернатива: нажать пресет.)
3. Появляется **подтверждающее окно**: 👤 Иванов А.П. · 📅 14–16.05 · 📊 ВА + Телематика.
4. Нажать [✓ Всё верно — показать отчёт].
5. Показать дашборд: карточка водителя, пробег 487 км, KPI (7 нарушений: 4 ВА / 3 телематика / 1 грубое).
6. Показать правую зону: трек маршрута на карте + график скорости с всплесками превышений.
7. Кликнуть строку «Курение за рулём (🔴 Грубое)» → видео выезжает справа.
8. Переключить на «По ТС» — показать что А777ВВ 77 имел двух водителей за период.
9. Показать badge [📷 CAM-03 offline] красный у Е902СТ 150.
   Сказать: «Камера не работала — часть нарушений могла не записаться. Система это видит.»
10. Вернуть «По водителям», кликнуть Иванов → мини-дашборд → [Открыть полный отчёт].
11. Сказать: «Один голосовой запрос — полный дашборд. Два среза: по водителям и по ТС.
    Логисты смотрят ТС и камеры. Служба безопасности — водителей и нарушения.
    Два среза: водители и ТС — разные роли смотрят своё.
    Два среза аналитики: по водителям и по ТС — каждая роль видит своё.
    Видео по клику — именно это просил Оздоев из ГПН. 800 машин, 5 стран.»

### 03:10–03:30 — Flow 4: Список заявок (резервный)

## Резервные ответы на вопросы жюри

**«Это реальные данные?»**
> Mock-данные основаны на реальных интервью с Фоминым (PepsiCo), Масловым (Балтика) и Оздоевым (ГПН, 800 ТС). Конкретные цифры — из их слов.

**«Как это интегрируется с SKAI?»**
> MVP показывает интерфейс. Бэкенд подключается к существующим API SKAI Online — events, telematics, video archive.

**«Чем отличается от FT-Cloud?»**
> FT-Cloud — видео рядом с картой. Мы — полный сценарий: событие → видео → телематика → диагностика → действие с изменением статуса. И всё на русском.

### Резервный flow — Экран заявок

1. Нажать "📋 Заявки" в сайдбаре
2. Показать KPIBar: "ТС в зоне риска", "Критичных" — реальные цифры
3. Найти через поиск "В345" → строка inc-002 (Петров)
4. Нажать "Открыть →" → переход на /incident/inc-002

**Говорить:** «Все события аккумулируются в одном месте. Диспетчер видит статусы,
может фильтровать и сразу переходить к разбору.»
