# Что мы строим

## Суть

Единое окно где телематика и видео работают вместе.
Диспетчер не переключается между системами — он работает в одном продукте.

---

## Экраны продукта

### MVP — четыре основных экрана

> **Приоритет идей:**
> 1. **Идея #1 (P0)** — Единое окно инцидента: телематика + видео синхронно, CAN-скорость + акселерометр + действия в одном экране
> 2. **Идея #2 (P0)** — Интерактивный отчёт: голос/текст (faster-whisper RU/KK/EN) → подтверждение → дашборд водителя или парка (fleet-report.json + fleet-vehicles.json)
> 3. Диспетчерское действие прямо из карточки (Экраны 1+2+3)

---

#### Экран 1 — Карта с машинами

Стартовый экран. Пользователь открывает продукт и видит карту.

Каждая машина — одна точка. Не набор терминалов. Маслов из Балтики
видел 3 машины как 5 точек — здесь так не будет.

Цвет маркера показывает уровень риска: красный / оранжевый / зелёный.
Клик на маркер — мини-карточка с типом последнего события и кнопкой перехода.

Справа от карты — лента активных событий за смену.
Фильтры: все / критичные / без видео.
Клик на событие в ленте → переход на карточку события.

---

#### Экран 2 — Список событий и заявок

Таблица всего потока за смену. Четыре цифры вверху:
машин в зоне риска, критичных, просрочено, закрыто.

Каждая строка: машина, водитель, тип события, источник данных
(телематика / видео / оба), есть ли видео, статус, ответственный.

Цвет левой границы — уровень риска.
Поиск по номеру машины или имени водителя.
Клик на строку → карточка события.

---

#### Экран 3 — Карточка события с телеметрией

Главный экран продукта. Открывается из карты или из списка.

**Верх:** машина, водитель, тип события, время, уровень риска, статус.

**Видео (левая часть):**
Два плеера синхронно — ADAS (дорога) и DMS (водитель).
Таймлайн ±30 секунд от момента события.
Если видео недоступно — объяснение причины и кнопка запроса архива.

**Телеметрия (центр):**
График скорости из CAN-шины и акселерометр.
Маркер двигается вместе с видео — смотришь нужный момент,
видишь точные данные в эту секунду.
Момент события — красная вертикальная линия.

**Действия (правая часть):**
- Запросить видео за период
- Создать заявку
- Сформировать отчёт
- Вызвать водителя

Статусы источников: камеры, телематика, связь, архив, калибровка.

---

#### Экран 4 — Диалог аналитики

Пользователь задаёт вопрос — система возвращает результат или документ.

Поле ввода и набор готовых запросов:
- «ДТП без срабатывания DMS за этот месяц»
- «Топ 5 водителей по резким торможениям»
- «Машины где камера не работает»
- «Резкие торможения выше 80 км/ч»
- «Покажи нарушения по ВА и телематике у Иванова за последние три дня»

**Ключевой поток (голос/текст → подтверждение → интерактивный отчёт):**
Диспетчер нажимает 🎤 и говорит (или печатает) запрос
→ faster-whisper large-v3 (open-source, RU 🇷🇺/KK 🇰🇿/EN 🇬🇧) расшифровывает речь в текст
→ система показывает подтверждающее окно (водитель, период, тип данных)
→ диспетчер подтверждает одним кликом
→ система строит дашборд: карточка водителя + ТС, пробег, KPI нарушений,
трек маршрута, график скорости, таблица ВА+телематика, видео по клику.

**Голосовой ввод на базе Whisper AI** — диспетчер не отрывает руки от других задач.
Whisper обрабатывает речь локально или через API, результат появляется в поле запроса.

Ответ: дашборд по водителю/периоду, список событий с переходом в карточку или PDF-отчёт.
Это не конструктор дашбордов — это быстрый ответ на конкретный вопрос диспетчера.

Источник: интервью Оздоев (ГПН): «Нажал нарушение — вылезло видео.» — подтверждённый рыночный запрос.

### Расширенные экраны — если успеем

---

#### Экран 5 — Интерактивный отчёт безопасного вождения (финал NL-потока)

**Это финальный экран сценария #2 — продолжение Экрана 4.**
NLU определяет режим результата автоматически:

**Режим В-1 — один водитель** (запрос: «Нарушения Иванова за три дня»):
Объединённая таблица: телематика + видеодетекции в одном месте.
Клик на строку → видео выезжает справа.
По умолчанию справа: трек маршрута + график скорости за период.
Грубые нарушения → взыскание / отстранение подрядчика (banner).

**Режим В-2 — весь парк** (запрос: «Все нарушения по компании за три дня»):
Переключатель «По водителям | По ТС» — разные роли смотрят по-разному:
Рейтинг водителей по числу нарушений (список с Score и badges ВА/тел/грубых).
Bar chart по дням: стопка ВА + Телематика.
Карта кластеров нарушений по парку.
Вкладка «По водителям»: список с Score + badges → мини-дашборд → [Открыть полный отчёт] → В-1.
Вкладка «По ТС»: ТС = актив компании, несколько водителей за период.
  — Статус камер: online / offline — критично (если CAM offline, часть нарушений не записана)
  — Водители за период: основной + подменный
  — Клик → мини-дашборд ТС с историей и камерами → [Открыть историю ТС]
Аудитория: логисты смотрят «По ТС», служба безопасности — «По водителям».

Источник — Оздоев (ГПН, 800 машин): «Нажал нарушение — вылезло видео.»
ГПН строит SMAP LPS 2.0 именно под эти два режима.

#### Экран 6 — Диспетчерский алерт

Из интервью Оздоева: «Резко затормозил — запрошено видео 15 секунд
вперёд и назад. Связка с диспетчерским центром.»

Сейчас диспетчерская служба есть, но звонки не производятся автоматически.

При критическом событии — алерт с видео ±15 секунд,
телеметрией момента и тремя кнопками:
Позвонить водителю / Создать заявку / Всё в порядке.
Таймер ожидания реакции диспетчера.

---

#### Экран 7 — Видеодосье рейса

Из брифа организаторов хакатона.

Весь рейс на одном экране: трек на карте с видеоотметками,
временная шкала где каждое событие — точка.
Клик на точку → видео именно этого момента.

Для разбора сложных ДТП где важна хронология всей поездки,
а не только момент события.

---

#### Экран 8 — Восстановление маршрута при РЭБ

Из брифа организаторов: «Снижение влияния РЭБ за счёт анализа видео.»

Когда GPS глушат — телематика теряет трек. Видео остаётся.
Система восстанавливает маршрут по анализу видеопотока:
дорожные знаки, разметка, повороты.

Показывает: фактический маршрут по видео рядом с GPS-треком.

## Как экраны связаны

```
Карта (1)
  → клик на маркер → Карточка события (3)
  → клик в ленте событий → Карточка события (3)

Список (2)
  → клик на строку → Карточка события (3)

Карточка (3)
  → кнопка «Отчёт» → Диалог аналитики (4)
  → кнопка «Назад» → откуда пришли (1 или 2)

Отчёт БВ (5)
  → клик на нарушение → Карточка события (3)

Досье рейса (7)
  → клик на точку → Карточка события (3)
```

## Два сценария для жюри

**Flow 1 — ДТП без камеры водителя (Петров):**
Красный маркер на карте → карточка → скорость 54→0 за 4 секунды,
пик акселерометра → нажимаем «Создать заявку» → статус изменился.

**Flow 2 — Нет видео (Сидоров):**
Пометка в ленте → карточка → «CAM-03 не передала данные» →
нажимаем «Запросить архив» → «Запрос создан, ожидайте 15–30 мин».
