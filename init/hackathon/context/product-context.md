## Главная цель MVP

Доказать: когда видео и телематика в одном сценарии, клиент быстрее понимает что произошло, почему и какое действие нужно.

```
Телематическое событие или видеоаларм
→ единая карточка ТС / инцидента
→ синхронное видео + CAN-скорость + акселерометр + трек / timeline
→ объяснение причины
→ действие: запрос видео, отчёт, звонок водителю, валидация или заявка
```

Источник: Олег Фомин, PepsiCo.

3 машины → 5 терминалов на карте. 1 ТС = 1 объект; внутри: телематика, видео, камеры, статусы.

### Кейс 4. Быстрое действие из единого окна
Источники: Фомин, Маслов, сравнение SKAI / FT-Cloud.  
После просмотра события — сразу выполнить действие без переключения экранов.

## Что НЕ делаем

- Отдельный дашборд «Здоровье парка»
- Полноценную превентивную диагностику ТС
- NL-to-SQL / NL-to-Chart как основной продукт
- Обучение водителей (другая команда)
- Реальный backend, CAN, авторизацию, Docker, CI/CD

## Что можно из темы 8 (только вспомогательно)

- Статус камер: работают / не работают
- Статус телематики: доступна / нет данных
- Статус архива: видео доступно / нужно запросить

## Бизнес-термины

| Термин | Значение |
|--------|----------|
| ТС | Транспортное средство |
| Alarm / Аларм | Событие видеоаналитики |
| DMS | Состояние водителя: засыпание, телефон, отвлечение |
| ADAS | Дорожные события: FCW, HMW, LDW |
| CAN | Источник достоверной скорости и параметров ТС |
| Timeline | Единая временная ось видео и телематики |

## Ключевой NL-сценарий (добавлен 16.05.2026)

### Сценарий: «Покажи нарушения по ВА и телематике у Иванова за последние три дня»

**Это идея #2 в продукте** (после единого окна видео+телематика).

Пользователь нажимает 🎤 и говорит (или печатает) запрос в свободной форме.
**Whisper AI** (OpenAI) расшифровывает речь в текст — диспетчер не отвлекается от экрана.
Система показывает **подтверждающее окно**: «Вот как я понял ваш запрос» —
с параметрами: водитель, период, тип данных (ВА + телематика).
После подтверждения — полный интерактивный отчёт.

**Что выводит система в отчёте:**
- Карточка водителя: имя, рейтинг безопасности, прогресс-бар
- Карточка ТС: госномер, марка, пробег за период, количество рейсов, время в рейсах
- KPI-статистика: всего нарушений / ВА / телематика / грубых
- Трек маршрутов за период на карте с цветными точками событий
- График скорости за период с пиками превышений и лимитом
- Объединённая таблица нарушений (ВА + телематика в одной таблице)
- Клик на строку → видео выезжает справа (killer feature Оздоева)

**Источник:** интервью Руслан Оздоев (ГПН, ~800 ТС):
«Как Sky Studio: нажал превышение — вылез трек. А тут нажал нарушение — вылезло видео.»

**Ключевая бизнес-логика:**
- «По водителям» — для службы безопасности: кто нарушает
- «По ТС» — для логистов: состояние автопарка, камеры, водители за период
- ТС = актив компании. 1 ТС за период → несколько водителей
- Статус камер: online/offline влияет на полноту данных

**Почему это идея #2 в продукте:**
- Объединяет два источника (ВА + телематика) в одном запросе
- Голосовой ввод (Whisper AI) — диспетчер говорит, не печатает
- Подтверждающее окно убирает риск «система поняла не так»
- NL-интерфейс снижает порог входа: не нужно знать фильтры
- Видео по клику — прямая реализация запроса Оздоева (ГПН)
- ГПН строит SMAP LPS 2.0 именно под этот сценарий — подтверждённый спрос

**Технологический стек — речь и анализ текста:**

STT (речь → текст):
- **faster-whisper large-v3** — локально, open-source (Whisper от OpenAI, оптимизирован)
- Поддерживает русский 🇷🇺, казахский 🇰🇿, английский 🇬🇧 нативно (обучена на всех трёх)
- Модель ~3GB, запускается на CPU/GPU без внешних зависимостей
- Интеграция: кнопка 🎤 → запись WAV → faster-whisper → текст в поле

NLU (текст → структурированный запрос):
- **Groq API + LLaMA 3.3 70B** — бесплатный тариф, 300+ токенов/сек, практически мгновенно
- Задача простая: запрос → JSON {driver, period, data_types}
- Пример: «Покажи нарушения Иванова за три дня» → {driver: "Иванов", days: 3, types: ["va","tel"]}
- Альтернатива offline: **Ollama + LLaMA 3.1 8B** — полностью локально, бесплатно

Почему эти технологии:
- Оба — open-source модели, нет vendor lock-in
- Разворачиваются на серверах клиента (ГПН, PepsiCo)
- Стоимость API: Groq ~$0 на бесплатном тарифе, или ~$0.0009/1K токенов платно
- Казахский язык: Whisper large-v3 обучен на kk нативно; LLaMA 3.3 70B читает казахский
